## 2. 产品背景

企业会议中经常出现任务口头布置后无人记录、责任人不清、执行状态不可追踪、会议结束后任务遗漏等问题。该产品通过 Android App 进行会议录音或录音文件上传，调用第三方 AI 能力完成语音转文本、会议摘要和任务抽取，并在人工审核后发布任务，实现从会议记录到任务派发、执行确认、附件回传和结果归档的闭环。

产品同时提供个人备忘提醒能力，帮助业务人员记录临时事项和未来待办，降低日常工作遗漏风险。

## 3. 产品目标

1. 支持企业会议录音、上传、AI 转写、AI 摘要和任务抽取。
2. 支持会议发起人或授权审核人对 AI 结果进行修改、驳回、重跑和发布。
3. 支持任务自动创建、分配、提醒、确认、进度提交、执行反馈、附件上传和完成归档。
4. 支持超级管理员维护公司组织架构、人员账号和角色权限。
5. 支持会议原始录音、转写文本、AI 处理结果、人工修改记录、任务发布记录等材料留存，便于追溯。
6. 提供个人备忘提醒功能，作为辅助但必须具备的能力。

## 4. 产品范围

### 4.1 范围内

- Android 手机 App。
- App 内会议录音。
- 上传手机中已有录音文件。
- 云端存储会议录音、转写文本、任务、附件等数据。
- 云端调用第三方 AI API，优先按需求使用阿里 AI 能力。
- 会议转写、摘要、任务抽取、审核、发布、执行跟踪。
- 企业组织架构、人员账号、权限角色管理。
- 个人备忘和提醒。

## 5. 用户角色与权限

| 角色 | 说明 | 关键权限 |
| --- | --- | --- |
| 超级管理员 | 企业最高系统管理角色 | 管理公司组织架构、部门、岗位、人员账号、账号启停、角色权限分配 |
| 会议发起人 | 创建会议并发起录音或上传录音的人 | 创建会议、上传录音、查看 AI 结果、审核任务、授权他人审核、发布任务 |
| 授权审核人 | 由会议发起人授权的审核人员 | 审核会议转写文本、修改内容、驳回、重跑 AI、审核任务列表、发布任务 |
| 任务执行人员 | 被分配任务的企业成员 | 接收任务提醒、确认任务、提交任务进度、更新执行状态、上传附件、完成任务 |
| 普通用户 | 使用个人备忘功能的人员 | 创建个人备忘、设置提醒、管理自己的提醒事项 |
| AI 服务 | 第三方 AI 能力提供方 | 语音转文本、摘要生成、任务抽取、低置信度识别 |

## 6. 核心使用场景

### 6.1 企业会议任务派发

会议发起人使用 Android App 进行会议录音，或在会议结束后上传已有录音文件。系统联网后将录音上传至云端，调用第三方 AI API 转写为文本，并进一步生成会议摘要和任务列表。审核人员检查文本和任务，确认无误后发布给执行人员。

### 6.2 跨部门任务执行跟踪

任务发布后，执行人员收到类似微信消息提醒的弹窗通知。执行人员确认任务后，在 App 内自行提交当前任务进度、更新执行状态，上传文件或图片作为执行证明。会议发起人或审核人员可查看任务执行进度。

### 6.3 会议内容溯源

当任务内容存在争议时，审核人员或有权限用户可查看会议原始录音、AI 转写文本、AI 处理结果、人工修改记录和最终发布任务，追溯任务来源。

### 6.4 个人备忘提醒

普通业务人员可创建个人待办事项，设置提醒时间。系统按提醒时间弹窗提醒，若用户未自定义提醒策略，则使用默认提醒时间。

### 6.5 企业组织与账号初始化

超级管理员登录后创建公司组织架构，维护部门、岗位和人员账号，并为用户分配角色权限，确保会议任务可以准确分配到对应人员。

## 7. 功能需求

### 7.0 需求 ID 与优先级说明

| 优先级 | 定义 |
| --- | --- |
| P0 | MVP 必须实现，缺失会导致主流程不可用 |
| P1 | 重要增强能力，建议首版实现或紧随 MVP 实现 |
| P2 | 后续优化能力，不影响 MVP 主流程闭环 |

### 7.1 超级管理员与组织账号管理

#### 7.1.1 组织架构管理 [FR-ORG-001] [P0]

- 超级管理员可创建、编辑、删除公司部门。
- 支持多级部门结构，用于匹配企业内部组织关系。
- 部门信息至少包括部门名称、上级部门、部门负责人、状态。
- MVP 阶段支持手动创建组织架构，后续支持 Excel 批量导入。

#### 7.1.2 人员账号管理 [FR-ORG-002] [P0]

- 超级管理员可创建人员账号。
- 超级管理员可编辑人员姓名、手机号、所属部门、岗位、角色、账号状态。
- 支持启用、停用人员账号。
- 人员账号应与组织架构关联。

#### 7.1.3 角色权限管理 [FR-ORG-003] [P0]

- 超级管理员可为人员分配角色。
- 系统内置角色包括超级管理员、会议发起人、授权审核人、任务执行人员、普通用户。
- 同一人员可拥有多个角色。

### 7.2 会议录音与上传

#### 7.2.1 App 内录音 [FR-REC-001] [P0]

- 用户可在 Android App 内创建会议并开始录音。
- 录音过程中应展示录音状态和录音时长。
- 无网络时不可使用 App 内录音能力。
- 录音完成后，文件上传至云端保存。

#### 7.2.2 录音文件上传 [FR-REC-002] [P0]

- 用户可上传手机中由其他软件录制的音频文件。
- 首期支持 `mp3`、`m4a`、`wav` 格式。
- 单个录音文件最大支持 200MB。
- 无网络时不可上传；有网络后可上传本地录音文件。
- 上传完成后进入 AI 转写流程。

### 7.3 AI 转写与处理

#### 7.3.1 语音转文本 [FR-AI-001] [P0]

- 系统调用第三方 API 进行语音转文本，第三方 API 优先使用阿里能力。
- 转写必须联网，离线状态不支持转写。
- AI 转写可在后台进行，用户不需要停留在当前页面等待。
- 转写完成后通知会议发起人或审核人员。

#### 7.3.2 摘要与任务抽取 [FR-AI-002] [P0]

- 系统基于转写文本调用第三方 API 生成会议摘要。
- 系统基于转写文本自动抽取任务列表。
- 任务抽取字段至少包括任务标题、任务内容、执行人员或执行部门、截止时间、来源会议、附件要求。

#### 7.3.3 低置信度提示 [FR-AI-003] [P1]

- 当 AI 转写或任务抽取结果置信度低于系统配置阈值时，应在审核文本页面最上方使用红色字体提示审核人。
- 提示文案示例：`AI 识别置信度较低，请重点核对转写文本和任务列表。`

#### 7.3.4 重跑机制 [FR-AI-004] [P1]

- 在审核文本阶段触发重跑时，系统重新执行语音转文本，并重新生成任务列表。
- 在审核任务阶段触发重跑时，系统基于当前审核后的文本重新生成任务列表。
- 重跑结果无需与历史版本进行对比。
- 重跑过程中应保留原始录音文件，保证可溯源。

### 7.4 审核与发布

#### 7.4.1 审核人员范围 [FR-AUDIT-001] [P0]

- 审核人员为会议发起人或会议发起人授权的人员。
- MVP 阶段单场会议仅支持一个授权审核人，后续可扩展多人协作审核。
- 未经授权的人员不可审核和发布该会议任务。

#### 7.4.2 审核界面 [FR-AUDIT-002] [P0]

- 审核人员可在审核界面播放会议录音。
- 审核人员先审核 AI 转写并处理后的文本。
- 若文本有问题，审核人员可手动修改文本。
- 修改文本后，可通过 AI 重新生成任务列表。
- 若文本无问题，审核人员进入任务列表审核。
- 任务列表采用列表展示。
- 点击任务条目可展开查看任务详情。
- 每条任务下方提供三个按钮：`删除`、`修改`、`发布`。
- 审核人员可修改任务标题、任务内容、执行人员或部门、截止时间、附件要求。

#### 7.4.3 驳回 [FR-AUDIT-003] [P0]

- 审核人员可点击驳回。
- 驳回后会议进入已驳回状态。
- 已驳回内容不发布给执行人员。

#### 7.4.4 发布 [FR-AUDIT-004] [P0]

- 审核人员经修改或未修改后，审核通过即可发布。
- 发布后任务分配给对应执行人员。
- 任务发布时触发执行人员提醒。

### 7.5 任务流转

#### 7.5.1 会议处理状态 [FR-FLOW-001] [P0]

| 状态 | 说明 | 触发条件 |
| --- | --- | --- |
| 待处理 | 录音已上传，等待 AI 处理 | 录音上传成功 |
| AI 处理中 | 系统正在转写、摘要、抽取任务 | AI 处理开始 |
| 待审核 | AI 已完成文本和任务生成，等待审核 | AI 处理完成 |
| 已驳回 | 审核人员驳回 | 审核人点击驳回 |
| 待发布 | 审核人员审核通过，任务具备发布条件 | 审核人员经修改或未修改后点击通过 |
| 已发布 | 任务已发布给执行人员 | 审核人员发布任务 |
| 已归档 | 会议及任务闭环完成 | 关联任务全部完成并归档 |

#### 7.5.2 任务执行状态 [FR-FLOW-002] [P0]

| 状态 | 说明 | 触发条件 |
| --- | --- | --- |
| 待确认 | 任务已发布，等待执行人员确认 | 任务发布成功 |
| 执行中 | 执行人员已确认并开始处理 | 执行人员点击确认 |
| 已完成 | 执行人员完成任务并提交结果 | 执行人员提交完成 |

### 7.6 任务提醒

#### 7.6.1 分配提醒 [FR-REMIND-001] [P0]

- 任务分配到执行人员时，系统立即提醒执行人员。
- 提醒方式为类似微信消息提醒的弹窗提醒。

#### 7.6.2 后续提醒 [FR-REMIND-002] [P0]

- 执行人员可自行设置任务提醒时间。
- 若执行人员未设置提醒时间，系统使用默认提醒时间。
- 默认提醒时间为截止时间前 24 小时提醒一次，截止时间前 1 小时再次提醒。

### 7.7 任务进度追踪、执行跟踪与附件上传

#### 7.7.1 进度提交与执行反馈 [FR-TRACK-001] [P0]

- 执行人员可确认收到任务。
- 执行人员可更新任务状态为执行中。
- 执行人员可自行提交当前任务进度。
- 任务进度支持填写进度百分比、进度说明和提交时间。
- 任务进度可多次提交，系统按时间记录每次进度变化。
- 会议发起人、授权审核人和任务相关人员可查看任务进度记录。
- 执行人员完成任务后可提交完成反馈。

#### 7.7.2 附件上传 [FR-TRACK-002] [P0]

- 执行人员可上传文件、图片作为任务执行证明。
- 支持多选上传。
- 上传过程显示上传进度条。
- 附件单文件最大 50MB，单任务附件总量最大 500MB。
- 附件与对应执行记录关联保存。

### 7.8 会议溯源与文件留存 [FR-TRACE-001] [P0]

- 系统需保存会议原始录音。
- 系统需保存 AI 转写文本。
- 系统需保存 AI 摘要和任务抽取结果。
- 系统需保存人工修改后的文本和任务。
- 系统需保存任务发布记录、执行记录、附件和提醒记录。
- 会议相关文件统一关联到会议，便于后续追溯。

### 7.9 个人备忘提醒 [FR-MEMO-001] [P0]

- 用户可创建个人备忘事项。
- 备忘事项支持标题、内容、提醒时间。
- 用户可设置自定义提醒时间。
- 若未设置自定义提醒时间，系统使用默认提醒时间。
- 默认提醒时间为事项时间前 24 小时提醒一次，事项时间前 1 小时再次提醒。
- 到达提醒时间时，系统以弹窗方式提醒用户。
- 个人备忘数据仅本人可见。

## 8. 数据实体关系

### 8.1 实体定义

| 实体 | 说明 | 关键字段 |

| 执行记录 | 执行人员对任务的进度和反馈记录 | 记录 ID、任务 ID、执行人 ID、状态、进度百分比、进度说明、反馈内容、提交时间 |
| 附件 | 执行记录中的文件或图片 | 附件 ID、执行记录 ID、文件类型、文件地址、大小、上传时间 |
| 提醒 | 任务或备忘的提醒配置与记录 | 提醒 ID、关联对象类型、关联对象 ID、用户 ID、提醒时间、提醒状态 |
| 个人备忘 | 用户个人提醒事项 | 备忘 ID、用户 ID、标题、内容、提醒时间、状态 |

### 8.2 关联关系

- 一个企业包含多个部门和多个用户。
- 一个部门可包含多个子部门和多个用户。
- 一个会议由一个会议发起人创建，MVP 阶段可指定一个授权审核人，后续可扩展多个授权审核人。
- 一个会议可关联多个会议文件，包括原始录音、转写结果文件、摘要结果文件。
- 一个会议可生成多个任务。
- 一个任务必须关联一个会议。
- 一个任务可关联一个执行人员或一个执行部门。
- 一个任务可产生多条执行记录。
- 一条执行记录可关联多个附件。
- 一个任务可关联多个提醒。
- 一个个人备忘可关联一个或多个提醒。

## 9. 性能与兼容性要求

| 类别 | 要求 |
| --- | --- |
| AI 转写耗时 | 支持后台实时转写，用户可离开当前页面等待处理结果 |
| 录音文件大小 | 单个录音文件最大 200MB |
| 离线能力 | 无网络时不能使用 App 内录音；可使用手机其他软件录音，联网后上传 |
| AI 处理 | 转写、摘要、任务抽取必须联网 |
| 兼容性 | 仅适配 Android 手机 |

## 10. 安全与合规要求

- 业务数据存储在云端。
- 云端数据必须加密存储。
- 会议录音、转写文本、任务内容、附件属于企业敏感数据，应按用户权限控制访问。
- 非会议发起人、授权审核人、任务相关执行人员、超级管理员不可查看无关会议和任务数据。
- 用户账号停用后不可登录系统。

## 12. 异常与边界条件表

| 场景 | 条件 | 系统预期行为 |
| --- | --- | --- |
| App 内录音时无网络 | 用户点击开始录音 | 开始录音按钮禁用或点击后 Toast 提示 `无网络，请联网后录音` |
| 录音过程中网络中断 | 用户正在 App 内录音 | 继续录音，录音结束保留在本地，提示 `网络异常，录音结束后需联网上传`；恢复网络后自行将录音进行上传 |
| 上传录音时无网络 | 用户选择录音文件并点击上传 | 阻止上传，Toast 提示 `无网络，请联网后上传` |
| 上传录音超过大小限制 | 单个录音文件大于 200MB | 前端拦截，提示 `录音文件不能超过200MB` |
| 上传录音格式不支持 | 文件格式不是 `mp3`、`m4a`、`wav` | 前端拦截，提示 `仅支持mp3、m4a、wav格式` |
| 上传录音中断 | 上传过程中网络断开或 App 切后台被系统中断 | 标记为上传失败，保留本地待上传记录，允许用户重新上传 |
| AI 转写时无网络 | 录音已上传但设备或服务网络不可用 | 会议保持待处理或 AI 处理中状态，提示 `网络异常，转写将在网络恢复后继续` |
| 第三方 AI API 调用失败 | 阿里 API 超时、限流或返回失败 | 标记 AI 处理失败，展示失败原因，提供重试入口 |
| AI 低置信度 | 第三方 API 返回置信度低于系统阈值 | 在审核文本页面顶部用红色字体提示审核人重点核对 |
| 审核文本阶段重跑 | 审核人员点击重跑 | 重新执行语音转文本，并基于新文本重新生成任务列表 |
| 审核任务阶段重跑 | 审核人员点击任务重跑 | 基于当前审核后的文本重新生成任务列表，不对比历史版本 |
| 未授权审核 | 非会议发起人或授权审核人进入审核页 | 阻止访问，提示 `无审核权限` |
| 审核人驳回 | 审核人员点击驳回 | 会议状态变为已驳回，不发布任务，不提醒执行人员 |
| 审核通过但未发布 | 审核人员点击通过但未点击发布 | 会议状态变为待发布，任务不推送给执行人员 |
| 发布任务缺少执行人 | 任务未选择执行人员或执行部门 | 阻止发布，提示 `请选择执行人员或执行部门` |
| 发布任务缺少截止时间 | 任务未设置截止时间 | 阻止发布，提示 `请设置任务截止时间` |
| 执行人员提交进度为空 | 未填写进度百分比和进度说明 | 阻止提交，提示 `请填写任务进度` |
| 进度百分比超出范围 | 进度小于 0 或大于 100 | 前端拦截，提示 `进度范围为0%-100%` |
| 非任务执行人员提交进度 | 未被分配该任务的用户提交进度 | 阻止提交，提示 `无任务操作权限` |
| 附件上传无网络 | 执行人员上传附件时无网络 | 阻止上传，提示 `无网络，请联网后上传附件` |
| 附件单文件超限 | 单个附件大于 50MB | 前端拦截，提示 `单个附件不能超过50MB` |
| 附件总量超限 | 单任务附件总量超过 500MB | 前端拦截，提示 `附件总量已达500MB上限` |
| 附件上传中断 | 上传过程中网络断开 | 标记附件上传失败，保留重试入口，不提交不完整附件 |
| 默认提醒时间已过 | 任务发布时间晚于默认提醒时间点 | 不补发已过期默认提醒，仅保留后续仍未过期提醒 |
| 用户自定义提醒早于当前时间 | 执行人员设置过去时间 | 阻止保存，提示 `提醒时间不能早于当前时间` |
| 账号被停用 | 已停用账号尝试登录或操作 | 阻止登录或操作，提示 `账号已停用，请联系管理员` |
| 组织架构删除存在关联人员 | 超级管理员删除仍有关联人员的部门 | 阻止删除，提示 `该部门下存在人员，请先调整人员归属` |
| 云端加密存储失败 | 文件或业务数据保存时加密失败 | 阻止保存，提示系统异常并记录错误日志，避免明文落库 |

## 13. MVP 建议

MVP 优先完成以下能力：

1. Android App 登录与账号体系。
2. 超级管理员管理组织架构和人员账号。
3. 会议创建、App 内录音、录音上传。
4. 云端保存录音文件，单文件最大 200MB。
5. 调用阿里第三方 API 完成语音转文本、摘要和任务抽取。
6. 审核人员播放录音、审核文本、修改文本、审核任务、修改任务、驳回或发布。
7. 发布任务后弹窗提醒执行人员。
8. 执行人员确认任务、提交当前任务进度、更新执行状态、上传文件或图片附件。
9. 保存会议、任务、执行记录、附件、提醒的关联关系。
10. 个人备忘提醒。

## 14. 验收标准

### 14.1 会议任务主流程

- 用户可在 Android App 创建会议并录音。
- 用户可上传不超过 200MB 的录音文件。
- 录音上传后系统可联网调用 AI 完成转写、摘要和任务抽取。
- AI 处理完成后会议进入待审核状态。
- 审核人员可播放录音、修改文本、重跑 AI、查看任务列表。
- 审核人员点击驳回后会议进入已驳回状态。
- 审核人员审核通过后会议进入待发布状态。
- 审核人员发布后任务进入执行人员的待确认列表。
- 执行人员收到类似微信消息提醒的弹窗。
- 执行人员可确认、执行、提交当前任务进度、上传附件并完成任务。

### 14.2 权限与管理

- 超级管理员可创建部门和人员账号。
- 超级管理员可启停账号并分配角色。
- 非授权人员不可审核或发布会议任务。
- 任务执行人员只能处理分配给自己的任务。

### 14.3 数据与安全

- 会议录音、转写文本、AI 结果、人工修改结果、任务、执行记录、附件、提醒均可按实体关系保存。
- 云端数据加密存储。
- 转写和 AI 处理必须联网。
- 无网络时 App 内录音不可用，但用户可在联网后上传其他软件录制的音频。

### 14.4 辅助备忘

- 用户可创建个人备忘事项。
- 用户可设置提醒时间。
- 用户未设置提醒时间时，系统按默认提醒时间提醒。
- 到达提醒时间后系统弹窗提醒。
- 个人备忘仅本人可见。
